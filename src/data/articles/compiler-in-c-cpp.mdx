# Quá trình biên dịch chương trình C/C++

## Trình biên dịch là gì?

Trình biên dịch (`compiler`) là một phần mềm giúp chuyển các đoạn mã, câu lệnh viết bằng ngôn ngữ lập trình sang một đoạn mã, chương trình khác, thường là ngôn ngữ bậc cao sang bậc thấp hơn, hay mã máy (các bit `0` và `1` mà máy tính, vi xử lý có thể hiểu được).

## Các quá trình biên dịch chương trình C:

Có thể coi quá trình này được chia thành 4 giai đoạn chính:

1. Tiền xử lý (`Preprocessor`).
2. Dịch sang mã Assembly (`Compiler`).
3. Dịch mã Assembly sang ngôn ngữ máy (`Assembler`).
4. Liên kết (`Linker`).

![compiler in c/c++](/assets/images/articles/compiler-in-c-cpp-1.jpeg)

### 1. Giai đoạn tiền xử lý

Nghe chữ “tiền" thôi mọi người cũng có thể đoán được hành động này, “tiền” tức là trước, nghĩa là trước khi compile source code, các chỉ thị bắt đầu với ký hiệu `#` sẽ được xử lý, cùng với việc các comments trong đoạn code sẽ được loại bỏ.

Khi ta `#include <stdio.h>`, tức là chương trình dịch sẽ tìm đến file `stdio.h` (file này thường có sẵn trong các trình biên dịch `C/C++`), và copy toàn bộ nội dung trong file này, và đặt vào vị trí thay cho cụm `"#include <stdio.h>"`. Vì vậy khi include các file này, ta có thể sử dụng các hàm, các macro được định nghĩa sẵn trong đó. Ngoài ra còn có các tiền xử lý như define, macro,.... mình sẽ nói kỹ hơn về bài sau!!

### 2. Giai đoạn Dịch sang mã Assembly

Sau khi `preprocessor`, lúc này ta đã có những file source code hoàn chỉnh (thường là những file có phần mở rộng là `.c`, `.cpp`, `.h`, `.hpp`,...), trình biên dịch sẽ dịch sang mã `Assembly` (hợp ngữ), một ngôn ngữ cấp thấp, rất gần với phần cứng.

### 3. Assembler

Khi đã có những mã `assembly` tương ứng, lúc này chương trình mới dịch tiếp sang mã máy, chúng là những file nhị phân (có phần mở rộng như `.o`, `.obj`,... tùy vào phiên bản `compiler` hay hệ điều hành). Thường những file như `hello.cpp` sẽ được dịch thành những file như `hello.o` (trên `UNIX`) hoặc `hello.obj` (trên `Windows`).

### 4. Linker

Khi đã có những mã máy (thường mỗi file code sẽ dịch thành một file mã máy), Làm cách nào để máy tính có thể chạy, thực thi được chương trình bởi có rất nhiều file mã máy? Điều đó được giải quyết trong quá trình linker, giúp liên kết các file mã máy trên lại thành một file thực thi duy nhất (phần mở rộng trên Windows là `.exe`, còn trên UNIX là `.out`).

Ngoài ra, trong giai đoạn `linker`, các file mã máy của các thư viện (`.lib`, `.a`) cũng được liên kết cùng, nên khi gọi những hàm không có trong các thư viện nào, chương trình dịch sẽ tìm sẽ lỗi và báo cho người lập trình biết ngay lập tức.

## Kết

Bài viết chỉ gồm những kiến thức cơ bản nhất khi học ngôn ngữ lập trình `C/C++`. Bài tiếp theo mình sẽ nói kỹ hơn về Tiền xử lý (`Proprocessor`) gồm chỉ thị `include`, `define` và các `macro`, mong được sự ủng hộ từ các bạn!
